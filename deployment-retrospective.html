‚ö†Ô∏è Major Challenges & Solutions

Challenge 1: Node.js Version Compatibility

Issue: Node.js 18+ required by modern packages but EC2 system only supported Node.js 16 due to GLIBC version constraints.

Solution:

Manual installation of Node.js 16.20.2 with proper PATH configuration and downgraded incompatible packages.

Prevention:

Use Docker containers or choose EC2 instances with newer OS versions that support Node.js 18+. Consider using Amazon Linux 2023 instead of Amazon Linux 2.

Challenge 2: Prisma Compatibility Issues

Issue: Prisma 6.x required Node.js 18+ and had WebAssembly compatibility issues with Node.js 16.

Solution:

Downgraded to Prisma 4.16.0 which is compatible with Node.js 16 and doesn't use problematic WebAssembly features.

Prevention:

Pin Prisma version in package.json and test compatibility before deployment. Use Docker for consistent environments.

Challenge 3: Vite Build System Issues

Issue: Vite 5+ required Node.js 18+ and had crypto.getRandomValues compatibility issues.

Solution:

Downgraded to Vite 4.5.0 which works with Node.js 16 and has stable crypto implementations.

Prevention:

Use .nvmrc files to lock Node.js versions. Test builds in CI/CD with target Node.js version.

Challenge 4: Asset Path Resolution

Issue: SCSS background-image paths not resolving correctly during Vite build process.

Solution:

Moved images to public directory and updated SCSS paths to use absolute paths from public directory.

Prevention:

Use Vite's asset handling features properly. Import images in components or use proper public asset paths from the start.

Challenge 5: CORS Configuration

Issue: API server not accessible from external clients due to CORS and network binding issues.

Solution:

Configured CORS properly and ensured server listens on all interfaces (0.0.0.0) instead of just localhost.

Prevention:

Configure CORS and network binding during development. Use environment variables for different deployment environments.

Challenge 6: Mobile Navigation Interference

Issue: Fixed-position mobile menu with high z-index blocking page interactions even when closed.

Solution:

Added pointer-events: none to closed menu and proper click-outside handling to prevent interference.

Prevention:

Test mobile interactions thoroughly. Use proper CSS techniques for overlay management and z-index stacking.

üîÑ Repetitive Error Patterns

Pattern 1: Engine Version Conflicts

Frequency: Occurred 5+ times throughout deployment

Error: "The engine 'node' is incompatible with this module. Expected version '>=18.0.0'. Got '16.20.2'"

Prevention Strategy:

Create a compatibility matrix document. Use .nvmrc and engines field in package.json. Consider using Docker for consistent environments.

Pattern 2: GLIBC Compatibility Issues

Frequency: Occurred 3+ times with different packages

Error: "version 'GLIBC_2.27' not found" or "version 'GLIBC_2.28' not found"

Prevention Strategy:

Use newer EC2 AMIs or Docker containers. Test package compatibility before deployment. Consider using pre-compiled binaries.

Pattern 3: WebAssembly Compatibility

Frequency: Occurred with Prisma and other modern packages

Error: "WebAssembly.Module(): invalid value type 'externref'"

Prevention Strategy:

Pin package versions that are compatible with target Node.js version. Use alternative packages or polyfills when necessary.

üìö Lessons Learned

1. Environment Consistency is Critical

Deployment environments should match development environments as closely as possible. Use Docker or containerization to ensure consistency.

2. Version Pinning is Essential

Pin all major dependencies to specific versions. Use package-lock.json or yarn.lock files. Test compatibility before deployment.

3. Infrastructure Planning Matters

Choose appropriate EC2 instance types and AMIs. Consider using newer OS versions that support modern Node.js versions.

4. CORS and Network Configuration

Configure CORS and network binding during development, not as an afterthought. Use environment variables for different deployment stages.

5. Asset Management Strategy

Plan asset handling strategy early. Use proper Vite asset handling or public directory approach consistently.

6. Mobile-First Testing

Test mobile interactions and responsive design thoroughly. Pay attention to z-index stacking and overlay management.

üöÄ Future Improvements

1. Containerization

Use Docker containers for consistent deployment environments. This eliminates Node.js version and GLIBC compatibility issues.

2. CI/CD Pipeline

Implement automated testing and deployment pipeline. Test compatibility in CI/CD before manual deployment.

Use proper environment configuration management. Separate development, staging, and production configurations.

4. Monitoring and Logging

Implement proper database migration and seeding strategies. Use Prisma migrations in production.

6. Security Hardening

Implement proper security measures: HTTPS, environment variable management, and security group optimization.
