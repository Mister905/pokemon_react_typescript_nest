üèóÔ∏è Phase 1: AWS EC2 Instance Setup

1Create EC2 Instance

üéØ Instance Configuration

Create an EC2 instance with sufficient resources for Docker containers. For the Pokemon app, we recommend t2.small or larger.

# AWS Console Steps: # 1. Go to EC2 Dashboard # 2. Click "Launch Instance" # 3. Choose Amazon Linux 2023 AMI (recommended over Amazon Linux 2) # 4. Select t2.small instance type (minimum for Docker) # 5. Create or select a key pair # 6. Configure security group (see next step) # 7. Launch instance
üí° Instance Size Recommendation

Use t2.small or larger. t2.micro (8GB) is too small for Docker builds with the Pokemon app dependencies.

2Configure Security Group

# Security Group Rules: # SSH (22) - Your IP only # HTTP (80) - 0.0.0.0/0 (for Nginx) # Custom TCP (3000) - 0.0.0.0/0 (for API) # Custom TCP (5173) - 0.0.0.0/0 (for Frontend) # Inbound Rules: Type: SSH, Protocol: TCP, Port: 22, Source: Your IP Type: HTTP, Protocol: TCP, Port: 80, Source: 0.0.0.0/0 Type: Custom TCP, Protocol: TCP, Port: 3000, Source: 0.0.0.0/0 Type: Custom TCP, Protocol: TCP, Port: 5173, Source: 0.0.0.0/0
üîí Security Explanation

SSH is restricted to your IP for security. HTTP ports are open to the internet for public access to your application.

üê≥ Phase 2: Docker Installation on EC2

4Install Docker on Amazon Linux 2023

# Update the system sudo dnf update -y # Install Docker sudo dnf install -y docker # Start Docker service sudo systemctl start docker sudo systemctl enable docker # Add ec2-user to docker group sudo usermod -a -G docker ec2-user # Log out and back in for group changes to take effect exit
üîÑ Group Changes

You need to log out and back in for the docker group membership to take effect. This allows you to run Docker commands without sudo.

üîß Troubleshooting Common Issues

‚ùå Issue: Docker Build Fails

Solution: Check disk space with df -h. If low on space, clean up with docker system prune -a.

‚ùå Issue: Services Won't Start

Solution: Check logs with docker-compose logs -f and verify all required files are present.

‚ùå Issue: Can't Access from Browser

Solution: Verify security group allows inbound traffic on ports 3000, 5173, and 80.

‚ùå Issue: Out of Memory

Solution: Upgrade to larger instance type (t2.medium or larger) or optimize Docker images.
