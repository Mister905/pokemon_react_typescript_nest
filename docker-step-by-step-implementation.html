ğŸ”§ Phase 3: Backend Dockerfile Creation

5Create API Dockerfile

# Navigate to API directory cd api # Create Dockerfile for the backend cat > Dockerfile << 'EOF' # Use latest LTS Node.js version FROM node:18-alpine # Set working directory WORKDIR /app # Copy package files first (for better caching) COPY package*.json ./ COPY yarn.lock ./ # Install dependencies RUN yarn install # Copy source code COPY . . # Generate Prisma client (if using Prisma) RUN npx prisma generate # Build the application RUN yarn build # Expose port EXPOSE 3000 # Start the application CMD ["yarn", "start:prod"] EOF
ğŸ” Key Points
  • Node.js 18: Latest LTS version with best compatibility
  • Latest dependencies: Use current stable versions
  • Multi-stage approach: Copy package files first for better caching
  • Prisma support: Includes Prisma client generation if needed

6Create API .dockerignore

ğŸ¨ Phase 4: Frontend Dockerfile Creation

7Create Client Dockerfile

# Navigate to client directory cd ../client # Create Dockerfile for the frontend cat > Dockerfile << 'EOF' # Build stage FROM node:18-alpine AS builder # Set working directory WORKDIR /app # Copy package files first (for better caching) COPY package*.json ./ COPY yarn.lock ./ # Install dependencies RUN yarn install # Copy source code COPY . . # Build the application RUN yarn build # Production stage FROM nginx:alpine # Copy built files from the build stage COPY --from=builder /app/dist /usr/share/nginx/html # Copy nginx configuration COPY nginx.conf /etc/nginx/conf.d/default.conf # Expose port EXPOSE 80 # Start nginx CMD ["nginx", "-g", "daemon off;"] EOF
ğŸ” Key Points
  • Multi-stage build: Builds with Node.js, serves with Nginx
  • Latest versions: Use current stable Node.js and dependencies
  • Nginx serving: Efficient static file serving
  • Port 80: Standard HTTP port

8Create Client Nginx Configuration

# Create nginx configuration for the client cat > nginx.conf << 'EOF' server { listen 80; server_name localhost; root /usr/share/nginx/html; index index.html; # Handle client-side routing location / { try_files $uri $uri/ /index.html; } # Cache static assets location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ { expires 1y; add_header Cache-Control "public, immutable"; } # Security headers add_header X-Frame-Options "SAMEORIGIN" always; add_header X-Content-Type-Options "nosniff" always; add_header X-XSS-Protection "1; mode=block" always; } EOF

ğŸ³ Phase 5: Docker Compose Configuration

10Create Development Docker Compose

# Navigate back to root directory cd .. # Create development docker-compose.yml cat > docker-compose.yml << 'EOF' version: '3.8' services: # NestJS Backend API api: build: context: ./api dockerfile: Dockerfile ports: - "3000:3000" environment: - NODE_ENV=development - DATABASE_URL=file:./dev.db volumes: - api_data:/app/dev.db healthcheck: test: ["CMD", "wget", "--spider", "http://localhost:3000/api"] interval: 30s timeout: 10s retries: 3 # React Frontend client: build: context: ./client dockerfile: Dockerfile ports: - "5173:80" depends_on: api: condition: service_healthy volumes: api_data: EOF
ğŸ” Key Features
  • Health checks: Ensures API is ready before starting frontend
  • Volume persistence: Database data survives container restarts
  • Port mapping: Frontend on 5173, API on 3000
  • Service dependencies: Frontend waits for API to be healthy

11Create Production Docker Compose

# Create production docker-compose.yml cat > docker-compose.prod.yml << 'EOF' version: '3.8' services: # NestJS Backend API api: build: context: ./api dockerfile: Dockerfile ports: - "3000:3000" environment: - NODE_ENV=production - DATABASE_URL=file:./dev.db volumes: - api_data:/app/dev.db healthcheck: test: ["CMD", "wget", "--spider", "http://localhost:3000/api"] interval: 30s timeout: 10s retries: 3 # React Frontend client: build: context: ./client dockerfile: Dockerfile ports: - "5173:80" depends_on: api: condition: service_healthy # Reverse Proxy (Optional) nginx: image: nginx:alpine ports: - "80:80" volumes: - ./nginx/nginx.conf:/etc/nginx/nginx.conf - ./nginx/default.conf:/etc/nginx/conf.d/default.conf depends_on: - api - client profiles: - production volumes: api_data: EOF

ğŸŒ Phase 6: Nginx Reverse Proxy Setup

12Create Nginx Directory and Configuration

# Create nginx directory mkdir -p nginx # Create main nginx configuration cat > nginx/nginx.conf << 'EOF' events { worker_connections 1024; } http { upstream api { server api:3000; } upstream client { server client:80; } server { listen 80; server_name localhost; # Frontend location / { proxy_pass http://client; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } # API location /api { proxy_pass http://api; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } } } EOF

13Create Nginx Default Configuration

# Create default nginx configuration cat > nginx/default.conf << 'EOF' server { listen 80; server_name localhost; # Frontend location / { proxy_pass http://client:80; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } # API location /api { proxy_pass http://api:3000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } } EOF
